## A. 繰り返し遊べる要素（残りの優先候補）

### A-2. デイリーチャレンジ 【次点・自己完結で実装可】
- 「全員が同じ配牌」を**日付シードの決定論的乱数**で実現（例：seed = YYYYMMDD）
- 現在の `shuffle()` / `dealHands()` / `pick6()` は `Math.random()` 依存 → **シード可能なPRNG（mulberry32等）に差し替えられる形**に一本化するのが肝
- 将来の「リプレイ／結果共有」の土台にもなる

### A-3. レート / ランク（自己ELO）＋ 難易度追加「MASKD MASTER」
- 対CPUなので実質「難易度別の自己ELO」。勝敗でレート増減、難易度で係数変更（ローカル完結）
- レビュー提案の**第4難易度「MASKD MASTER」**を追加するならここ。現行の `cpuEps`（うっかり率）を0＋思考をさらに厳密化（読みの深化）で表現
- ランキング化するなら A-4 と接続

### A-4. オンラインランキング 【要・事前設計】
- Firestore活用。着手前に決める判断が多い：
  - **不正対策**：クライアントスコアは自己申告になり得る（サーバ検証／リプレイ検証／統計的外れ値排除）
  - **セキュリティルール**：現状は書き込み専用。読み取り解禁時の範囲・レート制限
  - **表示名モデレーション**・**プライバシー**（現在は匿名。名前公開なら同意フロー見直し）
- 一番重い。A-2〜A-3を先に

### A-5. リプレイ（過去の対戦を再生）
- 対戦履歴に既にラウンド別詳細（`rounds`：親子・出した札・勝敗・得点）を保存済み
- 1手ずつ再生するビューを作れば実装可能。デイリー/共有とも相性良い

### A-6. AI解析「この場面の最善手」
- 既存の `cpuChooseCard` / `cpuEstimate` を流用し、各局面で最善手と実際の手を比較表示
- 初心者ヒント機能の発展形。学習価値が高い

### A-7. 実績のさらなる拡充・レア実績
- 現在18種。例：100勝・全属性で決戦勝利・特定コンボ達成 など。解除演出（トースト等）の強化も

## B. コード品質・保守性

> 方針：機能追加を優先しつつ内部を軽く整理。大規模リファクタは急がない。
> 単一ファイルは**意図的な制約**（自己完結・ビルド不要・GitHub Pagesにそのまま配置・カード画像はbase64埋め込み）。

- **B-1. ファイル分割**：`<script src>` で分割は可能（game.js / cpu.js / ui.js / tutorial.js / achievements.js / firebase.js / images.js(base64) など）。現行700KB超なので各機能の verify を用意してから段階的に。base64画像を `images.js` に隔離するだけでも見通し改善
- **B-2. CSSの共通化**：`.card` 系の重複を括り出し
- **B-3. アニメーション負荷**：`prefers-reduced-motion` は対応済み。旧端末向けに blur/box-shadow 多用箇所の軽量化余地
- **B-4. JSの責務分離**：ルール/UI/CPU/Firebase/保存/演出/チュートリアル/実績 が同居。B-1と合わせて段階的に

## メモ
- バランス検証はシミュレータ（本物のCPUロジック移植）で実施。ランダムbotではなく実ロジックで測る。scratchpad揮発のため必要なら再作成（`sim.js`/`cpumod.js`/`verify5.js` 方式。詳細は HANDOFF.md）
- 変更は毎回、実機（Playwright headless）で JSエラー0・通し完走・短い画面の見切れなしを確認してからデプロイ
- デプロイは main へ push → GitHub Pages（pages build and deployment）で自動反映
- バージョンは `GAME_VERSION` と `CHANGELOG` を必ず両方更新（詳細な引き継ぎは HANDOFF.md）
